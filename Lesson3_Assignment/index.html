Highlight #2 Local Repository v. Remote Repository

A remote repository services an entire organization. It resides on a remote server. It allows anyone person access to the system to be able to pull and push content that they have edited on a specific project. A local repository resides on a users actual machine. This allows staging and modification of a project and from their commit to changes for the general public.

Highlight #3 GIT

GIT is a form of version control and allows free and open source collaboration. It is an effective way to create snapshots of a current project so that everyone on a team knows who is working on what and in what fashion. It offers plenty of control for the advanced user but is also simple enough for a new user. Other benefits include no single point of failure and faster processing which is essential in a workforce. The three basic GIT commands are: add, commit and push. This can add content, save the content, and then allow the distribution of the content, which is the basic functionality of a version control system.
